## Purpose

This repository keeps short notes about collaboration workflows that can be verified in public history: opening a pull request, reviewing a documentation change, merging it, and checking the resulting commit. It contains documentation only, so each change remains easy to inspect.

## Collaboration rules

- Use pull requests for public README changes.
- Keep examples tied to actions that actually occurred on GitHub.
- Prefer one independently reviewable documentation change per pull request.
- Record merge evidence without creating empty or misleading activity.
- Do not treat a badge or contribution event as proof that a GitHub achievement will be awarded.
